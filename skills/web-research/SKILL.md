# Web research

When the user asks a research question, follow this process:

1. **Decompose** the question into 2–4 concrete sub-questions.
2. **Search** each sub-question with the web_search tool. Prefer primary and
   authoritative sources (official docs, standards bodies, original reporting).
3. **Cross-check** any important or surprising claim against at least two
   independent sources. Note disagreements explicitly.
4. **Synthesize** a concise answer that leads with the bottom line, then supporting
   detail.
5. **Cite** every non-obvious claim with a linked source, e.g. `[source](url)`.

Avoid presenting a single source as definitive for contested facts. If the
evidence is thin or conflicting, say so.
