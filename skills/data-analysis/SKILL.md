# Data analysis

1. **Understand the question** and what a good answer looks like (a number, a trend, a table).
2. **Inspect the data** before computing: shape, columns, types, a few sample rows, missing values.
3. **Compute with code** (the `python` tool) so results are reproducible — don't eyeball or
   guess numbers. Show the snippet you ran.
4. **Sanity-check** results against rough expectations; watch for unit, off-by-one, and
   null-handling mistakes.
5. **Report** the answer first, then the key supporting numbers, then caveats (sample size,
   assumptions, data quality).

Prefer the standard library / simple code. If you need a library that isn't installed,
install it with `sh` (pip) only if appropriate, or note the limitation.
